Simple Joomla Backup

Good to install your portal to another computer or on a web server and also useful for generating backup of your portal.

Provides speed, agility and comfort resulting in backup and migration of Joomla portals.
It's fast, it chose an inner class of PHP itself to compress (ZipArchive).

Using Joomla Simple Backup:

- Install the component com_simplejoomlabackup
- Click Components - simplejoomlabackup
- Click the Start button
- Wait and download two files
- Then Install on any computer with support for Joomla or on your own computer with another name.

Now just to restore:

- Create the database and import the .sql file for it
- Unzip the file. Create a folder in documentRoot and expand file to this
- Edit the configuration.php and adjust the database informations, $log_path and $tmp_path;

Now you can call in your browser.

Howto use the Joomla installer to restore your backup

- Unzip the Zip file generated by Simple Joomla Backup
- Download the Joomla 2.5 in the same version of your site
- Unzip the Joomla!
- Go to the unzipped folder and copy the sub-folder "installation" to the root of the backup folder
- Rename the file .sql from backup to joomla.sql
- Copy joomla.sql overwriting the existing to installation/sql/mysql
- Delete or rename the configuration.php file from the backup folder
- Compress the backup folder again and now you can restore your backup using Joomla installer

In case of failure in Backup

If for some reason the Simple Joomla Backup fails when creating the backup, the two best options I know are:
- Akeeba Backup - https://www.akeebabackup.com/download/akeeba-backup.html
   The configuration required is (to generate zip backup):
     Components - Akeeba Backup - Configuration
     Archiver engine - ZIP Format
- The other alternative is a manual backup (it does not dificult):
     Go to the cpanel file manager
     Select the folder for your site
     Click Compress
     Download the file generated
     Access phpMyAdmin
     Select the database from your site
     Export the database in the default format (.sql) and download
- Restoring a backup manual
     Ideally a backup is restored on a new site and new database
     Just go to the cpanel and unzip the zip files into a new folder
     Create a new database and go to phpmyadmin to import the script that initially generated
     Make the necessary adjustments in configuration.php if necessary

Errors

If after execute display the error message or similar:
Fatal error: Out of memory (allocated 273154048) (tried to allocate 200046879 bytes)
Enhance the memory_limit on your PHP. Send a support to your webhost.
